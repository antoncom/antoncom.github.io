Настройка прошивки для OpenWrt FAD Концепция FAD (Functionality, Adapter, Driver) предполагает модульную архитектуру. Каждый модуль строится на основе универсального блока ПО. Универсальный блок состоит из программного кода модуля (шаблона), переменных шаблона, модификаторов и настроек модификатора.

О системе
1. Есть источник данных (сигналов) и есть приёмник данных; 2. Источником данных может быть Драйвер, Адаптер, Функциональность, Пользователь, Константа, shell-команда; 3. Приёмником данных может быть либо файл- конфигурации, либо ОЗУ, то есть приёмник – это место хранения, полученных из источника данных. 4. При прохождении данных от источника к приёмнику они обрабатываются при помощи функций, называемых модификаторами. 5. Каждое данное с его модификаторами и свойствами модификатора обозначается переменной шаблона. Данные положения описывают универсальный строительный блок системы. На его основе делаются Адаптеры, Драйверы, Функциональности, а также модули UI, называемые виджетами: table_widget, modal_widget и др. Виджеты интерфейса, подписываются на события системной шины, и таким образом получают данные для обновления интерфейсных элементов.
• Тип ввода - строка, число, текст и т.п. (если тип источника=пользователь); • Валидатор - указываем функцию на основе Универсального метода валидации html-форм  (если тип источника=пользователь); • Логическая функция - данные от источника запрашиваются лишь при выполнении логического выражения; • Формула - данные от источника преобразовыва- ются по формуле; • Парсер - выбирает данные из источника согасно указанной функции разбора (грамматики); • Подписка - данные поступают от источника в приёмник лишь при возникновении события на системной шине; • Событие - инициирует указанное событие на сис- темной шине при поступлении данных от источника; • Автообновление - данные запрашиваются в источнике периодически с заданной частотой; • Область видимости - этот модификатор указы- вает в каких модулях системы видны переменные шаблона.
